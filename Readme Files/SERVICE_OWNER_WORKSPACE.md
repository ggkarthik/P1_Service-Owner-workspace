# Service Owner Workspace

## Overview

The **Service Owner Workspace** is an AI-native dashboard designed for business service owners who need to manage security risk at a service level. It provides a comprehensive view of cyber risk, remediation progress, business impact, and AI-powered recommendations in a cohesive, interactive layout.

> **Note:** The Service Owner Workspace is now fully integrated into the Microservices tab. Simply click on any service card in the Microservices tab to access its Service Owner Dashboard.

## Key Features

### ðŸŽ¯ Two-Layered Experience

The workspace provides two distinct views to cater to different needs:

#### 1. **High-Level View** (Executive Overview)
A strategic dashboard for quick assessment and decision-making:

- **Service Risk Score**: Large, prominent risk score (0-100) with business impact classification
- **Risk Trend Visualization**: 30-day trend chart showing risk evolution
- **Service Composition**: Visual diagram showing the service and its dependencies
- **Exposure Overview**: Bar charts showing vulnerability distribution by severity (Critical, High, Medium, Low)
- **Remediation Progress**: Key metrics including:
  - Number of breaches
  - Mean Time To Remediate (MTTR)
  - Assigned teams count
  - Open, In Progress, and Overdue vulnerabilities
- **Compliance Posture**: Circular progress indicator showing:
  - Adherence score
  - Applicable frameworks (PCI-DSS, SOC 2, ISO 27001, etc.)
  - Failed controls count
- **AI Insights & Recommendations**: AI-powered insights including:
  - Threat predictions and trends
  - Prioritized remediation recommendations
  - Business impact analysis
  - Next best actions

#### 2. **Granular View** (Detailed Analysis)
A comprehensive view for deep-dive analysis and action:

- **Vulnerability Table**: Detailed list of all vulnerabilities with:
  - Unique ID and CVE reference
  - Vulnerability type
  - Severity level (Critical, High, Medium, Low)
  - CVSS score
  - Affected package
  - Current status (Open, In Progress, Resolved, Mitigated)
  - Assigned team
  - Action buttons for remediation
- **Advanced Filtering**: Filter vulnerabilities by severity level
- **AI Recommendations**: Contextual recommendations based on selected vulnerabilities with:
  - Priority level
  - Business impact assessment
  - Effort estimation

## Usage

### Generating Fresh Data

```bash
# Generate new service owner data
python3 generate_service_owner_data.py

# Copy to React public directory (if needed)
cp data/generated/service_owner_data.json react-dashboard/public/data/
```

### Running the Dashboard

```bash
# Navigate to the React dashboard
cd react-dashboard

# Install dependencies (if not already done)
npm install

# Start the development server
npm start

# The dashboard will open at http://localhost:3000
# Navigate to: Dashboard â†’ Microservices
# Click on any service card to view its Service Owner Dashboard
```

**Alternatively, use the provided script:**

```bash
# Run the service owner workspace
./run_service_owner_workspace.sh
```

### Navigating the Workspace

1. **Access the Workspace**: Navigate to the Microservices tab in the main dashboard
2. **Select a Service**: Click on any service card to view its Service Owner Dashboard
3. **Toggle Views**: Switch between High-Level and Granular views using the toggle buttons
4. **Filter Vulnerabilities**: In Granular view, filter by severity (All, Critical, High, Medium, Low)
5. **Chat with AI**: Use the Conversational Copilot at the bottom to ask questions
6. **Take Action**: Click "View Details" on vulnerabilities to see remediation options
7. **Return to Services**: Click the "Back to Services" button to return to the Microservices overview
